HOW TO RUN AUDITS LOCALLY
================================================================================

SETUP (first time only)
------------------------
cd /Users/fela.sanu/Documents/code/website_audit_tool
source venv/bin/activate
cd gtm_verifier


TWO MODES
---------
1) FOREIGN SITE (prospect / cold audit) — no config file needed at all:
     python run.py --url https://prospect.com
   Runs the six site-agnostic audits: analytics_audit, consent_audit,
   network_audit, tag_inventory, seo, security_headers.

2) CLIENT SITE (we're under the hood) — YAML config with journeys:
     python run.py                          # config.yaml (Palm View test ground)
     python run.py --config client.yaml     # a specific client
   Runs the six audits PLUS every journey defined in the config's
   journeys: section, and verifies the expected GTM/GA4 IDs are live.

Useful flags:
  python run.py --list                     # show available audits + configured journeys
  python run.py shop cart                  # run specific audits/journeys only
  python run.py --export report.pptx       # PowerPoint instead of terminal
  python run.py --url https://staging.client.com --config client.yaml
                                           # client journeys against another env
  HEADLESS=1 python run.py ...             # no visible Chrome window


ONBOARDING A NEW CLIENT (no code changes, ever)
-----------------------------------------------
1. cp config.example.yaml client.yaml
2. Set site.base_url and (if known) tags.gtm_id / tags.ga4_id.
3. Set selectors.consent.accept_button only if the common-CMP auto-detection
   misses their banner.
4. Write journeys: for the flows that matter. Each journey is steps
   (goto / click / type / select_index / accept_consent / mark) plus
   expect blocks naming the dataLayer event, required fields
   (dot-notation), and optional regex patterns.
   Selector how-to: DevTools -> right-click element -> Copy -> Copy selector.
5. python run.py --config client.yaml

The full step/expect schema is documented in config.example.yaml and at the
top of journeys.py. palm_view_config.yaml is a complete worked example.


WEB FRONT END
-------------
flask --app webapp.app run          (from gtm_verifier/, HEADLESS=1 optional)
Open http://localhost:5000. Enter a URL for the public/authorized audits;
upload a client config YAML to also run its journeys and verify tag IDs.


NOTES
-----
- Events pushed by onclick handlers right before a page navigation ARE
  captured (persistent dataLayer log in core.py survives same-origin
  navigations). Cross-origin hops start a fresh log.
- Exit code is non-zero if any non-INFO check fails, so runs can gate CI.
- Known Palm View site gaps (expected FAILs, listed in palm_view_config.yaml):
  no select_item on product tiles, no customer_type on init, no contact
  form events.
